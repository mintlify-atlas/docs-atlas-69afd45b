Lakhua is a lightweight, blazing fast offline reverse geocoding library for India. Convert coordinates to location metadata with zero API costs and sub-millisecond latency.

## Installation

Choose your preferred language and install the SDK:

<CodeGroup>

```bash npm
npm install @aialok/lakhua
```

```bash bun
bun add @aialok/lakhua
```

```bash pip
pip install lakhua
```

```bash go
go get github.com/aialok/lakhua/libs/go
```

</CodeGroup>

All three SDKs have identical behavior and option signatures. Choose based on your project's language.

## Your first geocode lookup

Let's reverse geocode the coordinates for New Delhi:

<CodeGroup>

```typescript JavaScript/TypeScript
import { geocode } from "@aialok/lakhua";

const result = geocode(28.6139, 77.2090);
if (result) {
  console.log(result.city, result.state);
  // Output: New Delhi Delhi
}
```

```python Python
from lakhua import geocode

result = geocode(28.6139, 77.2090)
if result:
    print(result.city, result.state)
    # Output: New Delhi Delhi
```

```go Go
package main

import (
    "fmt"
    lakhua "github.com/aialok/lakhua/libs/go"
)

func main() {
    result := lakhua.Geocode(28.6139, 77.2090, nil)
    if result != nil {
        fmt.Println(result.City, result.State)
        // Output: New Delhi Delhi
    }
}
```

</CodeGroup>

<Info>
The first call loads JSON data into memory (~1.39 MB). All subsequent lookups are instant in-memory map operations.
</Info>

## Understanding the response

Each successful lookup returns location details with metadata:

<CodeGroup>

```typescript JavaScript/TypeScript
const result = geocode(12.9716, 77.5946);
// result => {
//   city: "Bengaluru",
//   state: "Karnataka",
//   district: "Bangalore North",
//   pincode: "560003",
//   matchedH3: "8560145bfffffff",
//   matchedResolution: 5,
// }
```

```python Python
result = geocode(12.9716, 77.5946)
# result ->
#   city: "Bengaluru"
#   state: "Karnataka"
#   district: "Bangalore North"
#   pincode: "560003"
#   matched_h3: "8560145bfffffff"
#   matched_resolution: 5
```

```go Go
result := lakhua.Geocode(12.9716, 77.5946, nil)
// result -> &lakhua.GeocodeResult{
//   LocationDetails: lakhua.LocationDetails{
//     City: "Bengaluru",
//     State: "Karnataka",
//     District: strPtr("Bangalore North"),
//     Pincode: strPtr("560003"),
//   },
//   MatchedH3: "8560145bfffffff",
//   MatchedResolution: 5,
// }
```

</CodeGroup>

**Response fields:**

- `city` — City name (always present)
- `state` — State name (always present)
- `district` — District name (optional, may be `null`/`None`/`nil`)
- `pincode` — Postal code (optional, may be `null`/`None`/`nil`)
- `matched_h3` — H3 cell index that matched
- `matched_resolution` — Resolution level of the match (4 or 5)

<Note>
Lakhua returns `null`/`None`/`nil` for invalid coordinates or locations outside the dataset coverage.
</Note>

## Common options

Customize lookup behavior with options:

### Enable debug mode

Track load and lookup timings:

<CodeGroup>

```typescript JavaScript/TypeScript
import { geocode } from "@aialok/lakhua";

const result = geocode(19.076, 72.8777, { debug: true });
// Prints:
// [lakhua][debug] loaded r5 store (123456 cells) in 45.23ms
// [lakhua][debug] lookup took 0.15ms
```

```python Python
from lakhua import geocode, GeocodeOptions

result = geocode(19.076, 72.8777, GeocodeOptions(debug=True))
# Prints load + lookup timings to stdout
```

```go Go
result := lakhua.Geocode(19.076, 72.8777, &lakhua.GeocodeOptions{
    Debug: true,
})
// Prints load + lookup timings to stdout
```

</CodeGroup>

### Disable fallback

By default, lakhua falls back to parent resolution (4) if no match exists at resolution 5. Disable this for exact-match-only behavior:

<CodeGroup>

```typescript JavaScript/TypeScript
import { geocode } from "@aialok/lakhua";

const result = geocode(28.6139, 77.2090, { fallback: false });
// Only checks resolution 5, no parent lookup
```

```python Python
from lakhua import geocode, GeocodeOptions

result = geocode(28.6139, 77.2090, GeocodeOptions(fallback=False))
# Only checks resolution 5, no parent lookup
```

```go Go
noFallback := false
result := lakhua.Geocode(28.6139, 77.2090, &lakhua.GeocodeOptions{
    Fallback: &noFallback,
})
// Only checks resolution 5, no parent lookup
```

</CodeGroup>

### Custom resolution

Change the H3 resolution for coordinate lookups (default is 5):

<CodeGroup>

```typescript JavaScript/TypeScript
import { geocode } from "@aialok/lakhua";

const result = geocode(28.6139, 77.2090, { resolution: 4 });
// Uses resolution 4 for initial lookup
```

```python Python
from lakhua import geocode, GeocodeOptions

result = geocode(28.6139, 77.2090, GeocodeOptions(resolution=4))
# Uses resolution 4 for initial lookup
```

```go Go
result := lakhua.Geocode(28.6139, 77.2090, &lakhua.GeocodeOptions{
    Resolution: 4,
})
// Uses resolution 4 for initial lookup
```

</CodeGroup>

## Direct H3 lookup

If you already have an H3 cell index, use the direct lookup method:

<CodeGroup>

```typescript JavaScript/TypeScript
import { geocodeH3 } from "@aialok/lakhua";

const result = geocodeH3("8560145bfffffff");
if (result) {
  console.log(result.city);
  // Output: Bengaluru
}
```

```python Python
from lakhua import geocode_h3

result = geocode_h3("8560145bfffffff")
if result:
    print(result.city)
    # Output: Bengaluru
```

```go Go
result := lakhua.GeocodeH3("8560145bfffffff", nil)
if result != nil {
    fmt.Println(result.City)
    // Output: Bengaluru
}
```

</CodeGroup>

<Warning>
Direct H3 lookups skip coordinate validation. Ensure your H3 indices are valid before calling.
</Warning>

## How it works

Lakhua uses a simple, efficient lookup process:

<Steps>

<Step title="Convert coordinates to H3">
Your `lat, lon` coordinates are converted to an H3 hexagonal cell index at resolution 5 (or your specified resolution).
</Step>

<Step title="In-memory lookup">
The H3 index is used as a key for an instant map lookup in the preloaded dataset.
</Step>

<Step title="Optional fallback">
If no match is found and fallback is enabled (default), lakhua walks up to the parent cell at resolution 4.
</Step>

<Step title="Return result">
Returns location metadata if found, or `null`/`None`/`nil` if no match exists.
</Step>

</Steps>

```text
lat, lon → H3 index (res 5) → in-memory lookup → fallback (res 4) → result / null
```

## Data coverage

The dataset covers major Indian cities and districts using H3 resolutions 4 and 5. Coverage is **not exhaustive** — rural or remote areas may return `null`.

<Info>
If you find a location that returns incorrect or missing results, [open an issue](https://github.com/aialok/lakhua/issues) with the coordinates.
</Info>

## When to use lakhua

Lakhua is ideal for:

- Backend services with high-frequency geocoding
- Batch processing of GPS coordinates
- Analytics workloads requiring consistent performance
- Air-gapped or offline environments
- Cost-sensitive applications

## When not to use lakhua

- You need rooftop-level or address-level precision
- You need global coverage beyond India
- You need live POI or business metadata
- You need hosted infrastructure with SLAs
