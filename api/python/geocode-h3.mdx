## Function signature

```python
def geocode_h3(
    h3_index: str,
    options: Optional[GeocodeOptions] = None,
) -> Optional[GeocodeResult]
```

Reverse geocodes an H3 cell index directly into India location metadata.

Uses the library's internal singleton geocoder. When fallback is enabled (default), parent resolutions are checked until the minimum supported resolution.

## Parameters

<ParamField path="h3_index" type="str" required>
  H3 cell index string (e.g., `"8560145bfffffff"`). Must be a valid H3 cell identifier.
</ParamField>

<ParamField path="options" type="GeocodeOptions" default="None">
  Lookup options to control fallback behavior and debug logging. See [GeocodeOptions](/api/python/types#geocodeoptions) for details.
</ParamField>

## Returns

<ResponseField name="result" type="GeocodeResult | None">
  Matched location details, or `None` when the H3 index is invalid or no match exists. See [GeocodeResult](/api/python/types#geocoderesult) for the complete structure.
</ResponseField>

## Behavior

- **Validation**: Returns `None` if the H3 index is invalid (checked using `h3.is_valid_cell()`)
- **Resolution handling**: Automatically detects the resolution of the input H3 cell
- **Fallback logic**: When enabled (default), checks parent cells starting from the input resolution down to resolution 4
- **Clamping**: Input resolutions are clamped to the supported range (4-5)
- **Performance**: Lookups are very fast (typically under 1ms) after initial data loading

## Resolution fallback

When `fallback=True` (default):

1. Checks the exact H3 cell at its native resolution
2. If no match, checks the parent cell at resolution 5 (if input was higher)
3. If still no match, checks the parent cell at resolution 4
4. Returns `None` if no match is found at any supported resolution

When `fallback=False`:

- Only checks the exact H3 cell (converted to its parent at the clamped resolution if needed)
- Returns `None` immediately if no exact match exists
