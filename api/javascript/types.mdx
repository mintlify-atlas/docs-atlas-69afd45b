## GeocodeOptions

Configuration options to tune lookup behavior for `geocode` and `geocodeH3`.

```typescript
interface GeocodeOptions {
  resolution?: number;
  fallback?: boolean;
  debug?: boolean;
}
```

### Fields

<ResponseField name="resolution" type="number" default="5">
  H3 resolution for coordinate conversion in `geocode(lat, lon)`. Valid range: 4-5. Higher resolutions provide more granular cells.
  
  This option only applies to the `geocode` function. It is ignored in `geocodeH3` since the resolution is determined from the input H3 index.
</ResponseField>

<ResponseField name="fallback" type="boolean" default="true">
  Enables parent fallback lookup from current resolution down to resolution 4.
  
  When `true`, if no match is found at the requested resolution, the lookup will check parent cells at lower resolutions until a match is found or resolution 4 is reached.
  
  When `false`, only the exact resolution is checked with no parent lookup.
</ResponseField>

<ResponseField name="debug" type="boolean" default="false">
  Prints timing logs for initial load and per-lookup access.
  
  When enabled, logs are written to the console showing:
  - Data loading time for each resolution
  - Lookup time for each resolution checked
  - Total time to find a match (or determine no match exists)
</ResponseField>

## GeocodeResult

Successful geocode result containing matched location details and metadata about the match.

```typescript
interface GeocodeResult extends LocationDetails {
  matched_h3: string;
  matched_resolution: number;
  city: string;
  state: string;
  district?: string;
  pincode?: string;
}
```

### Fields

<ResponseField name="matched_h3" type="string">
  The H3 cell index that produced the match. This may differ from the input H3 index if fallback was enabled and a parent cell was matched.
</ResponseField>

<ResponseField name="matched_resolution" type="number">
  The H3 resolution level at which the match was found (4 or 5). Useful for understanding whether fallback occurred.
</ResponseField>

<ResponseField name="city" type="string">
  City name associated with the matched location.
</ResponseField>

<ResponseField name="state" type="string">
  State name associated with the matched location.
</ResponseField>

<ResponseField name="district" type="string | undefined">
  District name if available. May be `undefined` for some locations.
</ResponseField>

<ResponseField name="pincode" type="string | undefined">
  Postal code (PIN code) if available. May be `undefined` for some locations.
</ResponseField>

### Example

```typescript
import { geocode } from "@aialok/lakhua";

const result = geocode(28.6139, 77.2090);

if (result) {
  console.log(result);
  // {
  //   city: "New Delhi",
  //   state: "Delhi",
  //   district: "Central Delhi",
  //   pincode: "110001",
  //   matched_h3: "8561891bfffffff",
  //   matched_resolution: 5
  // }
}
```

## LocationDetails

Normalized location details associated with an H3 cell.

```typescript
interface LocationDetails {
  city: string;
  state: string;
  district?: string;
  pincode?: string;
}
```

### Fields

<ResponseField name="city" type="string">
  City name for the location.
</ResponseField>

<ResponseField name="state" type="string">
  State name for the location.
</ResponseField>

<ResponseField name="district" type="string | undefined">
  District name if available. May be `undefined` for some locations.
</ResponseField>

<ResponseField name="pincode" type="string | undefined">
  Postal code (PIN code) if available. May be `undefined` for some locations.
</ResponseField>

<Note>
  `GeocodeResult` extends `LocationDetails` with additional metadata fields (`matched_h3` and `matched_resolution`).
</Note>

## ReverseGeoStore

In-memory reverse geocoding data store keyed by H3 index.

```typescript
type ReverseGeoStore = Record<string, LocationDetails>;
```

### Description

A dictionary mapping H3 cell indices (as strings) to their associated location details. This type is used internally by the data loader to cache resolution-specific lookup tables.

### Example

```typescript
const store: ReverseGeoStore = {
  "8561891bfffffff": {
    city: "New Delhi",
    state: "Delhi",
    district: "Central Delhi",
    pincode: "110001"
  },
  "866189b1fffffff": {
    city: "Mumbai",
    state: "Maharashtra",
    district: "Mumbai Suburban",
    pincode: "400001"
  }
};
```

<Note>
  Most users will not interact with `ReverseGeoStore` directly. It is primarily used by advanced users implementing custom data loaders.
</Note>

## Constants

The JavaScript SDK exports several constants that define the supported H3 resolution levels and data file configuration.

### Resolution constants

```typescript
const MIN_RESOLUTION = 4;
const MAX_RESOLUTION = 5;
const DEFAULT_RESOLUTION = 5;
const SUPPORTED_RESOLUTIONS = [4, 5] as const;
```

<ResponseField name="MIN_RESOLUTION" type="number" value="4">
  Minimum supported H3 resolution for lookup. This is the coarsest resolution level supported by lakhua.
</ResponseField>

<ResponseField name="MAX_RESOLUTION" type="number" value="5">
  Maximum supported H3 resolution for lookup. This is the finest resolution level supported by lakhua.
</ResponseField>

<ResponseField name="DEFAULT_RESOLUTION" type="number" value="5">
  Default H3 resolution used for `geocode(lat, lon)` coordinate conversion when no resolution option is specified.
</ResponseField>

<ResponseField name="SUPPORTED_RESOLUTIONS" type="readonly [4, 5]">
  Array of all supported resolution levels. This defines which resolution data files are preloaded into memory.
</ResponseField>

### Data file constants

```typescript
const DATA_DIR_NAME = "data";
const DATA_FILE_PREFIX = "reverse_geo_";
```

<ResponseField name="DATA_DIR_NAME" type="string" value='"data"'>
  Relative folder name where reverse geocoding JSON files are stored within the package.
</ResponseField>

<ResponseField name="DATA_FILE_PREFIX" type="string" value='"reverse_geo_"'>
  Prefix used by reverse geocoding data files. Data files follow the pattern `reverse_geo_{resolution}.json`.
</ResponseField>

### Example usage

```typescript
import { 
  MIN_RESOLUTION, 
  MAX_RESOLUTION, 
  DEFAULT_RESOLUTION,
  SUPPORTED_RESOLUTIONS,
  geocode 
} from "@aialok/lakhua";

console.log(`Supported resolutions: ${MIN_RESOLUTION}-${MAX_RESOLUTION}`);
// Output: Supported resolutions: 4-5

// Use a custom resolution within the supported range
const result = geocode(28.6139, 77.2090, {
  resolution: MIN_RESOLUTION  // Use coarsest resolution
});
```

<Note>
  These constants are useful when building custom geocoding workflows or validating user input for resolution values.
</Note>
